This document defines the roles, responsibilities, and expectations of maintainers of the Gojinn project.

Gojinn is governed under a **BDFL (Benevolent Dictator for Life)** model.

### Maintainers

Maintainers are trusted contributors appointed by the Founder.

They are expected to:

- Review pull requests
- Enforce design invariants and governance rules
- Provide technical guidance to contributors
- Help triage issues and discussions
- Uphold the projectâ€™s scope boundaries

Maintainers **do not override** the Founder on matters of:
- Core invariants
- Security model
- Project direction
